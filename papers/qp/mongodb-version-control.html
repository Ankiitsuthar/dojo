Long Questions

Attempt any three:

  1. You are working with a MongoDB database called social_media with collections posts, users, and comments. Perform the following tasks:

    1. Insert five documents into the posts collection with fields: content (string), author (string), likes (number), tags (array of strings), createdAt (date)
    2. Find all posts containing the tag "technology" and sort by likes in descending order
    3. Update all posts with more than 100 likes by adding a field trending set to true
    4. Calculate the average number of likes per author
    5. Delete all posts older than 30 days

  2. You are working with a MongoDB database called blog that contains the collections articles, authors, and comments. Perform the following tasks:

    1. Insert five documents into the articles collection with the fields: title (string), content (string), author (string), tags (array of strings), createdAt (date)
    2. Create a text index on the title and content fields of the articles collection.
    3. Find all articles containing the word "JavaScript" in either the title or content fields, sorted by createdAt in descending order.
    4. Update all articles containing the word "database" in the content field by adding a new field category with the value "Database".
    5. Delete all articles that do not contain any tags

  3. Imagine you are working on the clone of an open-source project in your local system. How will you achieve the following tasks using Git CLI commands:

    1. Set remotes for your forked remote (origin) and the upstream remote (upstream).
    2. Fetch all feature branches from the upstream remote.
    3. Create a feature branch feat6 linked to upstream/feat6.
    4. Stage and commit a change with the message "Completed Task 6" on the feat6 branch.
    5. Push the feat6 branch to your forked remote

  4. Imagine you are working on a feature branch and want to clean up your commit history by squashing multiple commits into one. How will you achieve the following tasks using Git CLI commands:

    1. View the commit history to identify the commits to squash.
    2. Start an interactive rebase for the last 4 commits.
    3. Squash the last 3 commits into the first commit, keeping the first commit's message.
    4. Force-push the squashed branch to the remote repository.
    5. Verify the commit history after squashing
