Multiple Choice Questions

  1. How do you sort documents in descending order by age?

    b) db.users.find().sort({ age: -1 })

    In MongoDB, sorting in descending order is done using -1, while ascending order uses 1.

  2. What operator updates if document exists, inserts if not?

    c) $upsert

    The $upsert parameter in update operations allows MongoDB to create a new document if no document matches the query criteria.

  3. Which operator matches values in an array?

    b) $in

    The $in operator selects documents where the value of a field equals any value in the specified array.

  4. What tool is used to import JSON/CSV data?

    a) mongoimport

    mongoimport is the command-line tool for importing JSON, CSV, or TSV files into MongoDB.

  5. What is the minimum number of nodes needed for a replica set?

    c) 3

    A replica set requires at least 3 nodes: one primary and two secondary nodes for proper failover and redundancy.

  6. What is the maximum number of indexes per collection?

    b) 64

    MongoDB limits the number of indexes per collection to 64 to maintain performance and resource management.

  7. You need to temporarily switch tasks but aren't ready to commit. What should you do?

    b) Use git stash

    git stash temporarily saves your changes without committing them, allowing you to switch tasks and come back later.

  8. You need to modify your last commit message. What command should you use?

    a) git commit --amend

    git commit --amend allows you to modify the most recent commit message without creating a new commit.

  9. You accidentally committed to main branch instead of feature branch. What's the best action?

    c) git reset to move commit to correct branch

    Using git reset allows you to undo the commit while keeping the changes, which you can then commit to the correct branch.

  10. What's the purpose of SSH keys in Git?

    b) Secure authentication with remote

    SSH keys provide secure authentication between your local machine and remote Git repositories without requiring password entry.

Short Questions

1. What is a capped collection in MongoDB?

A capped collection is a fixed-size collection in MongoDB that maintains insertion order and automatically removes the oldest documents when the size limit is reached.

Example:

db.createCollection("logs", {
    capped: true,
    size: 5242880,  // 5MB
    max: 5000       // Maximum 5000 documents
});

Commonly used for log files, audit trails, or any scenario where you only need to keep the most recent N documents.

2. Explain the concept of Sharding in MongoDB

Sharding is MongoDB's approach to horizontal scaling by distributing data across multiple machines.

Sharding strategies:

  • Range-based Sharding: Divides data into ranges based on the shard key value
  • Hash-based Sharding: Uses hashed values of the shard key for even distribution
  • Zone-based Sharding: Associates specific ranges of shard key values with specific shards

Each strategy has its use case. Range-based is good for range queries, hash-based for random distribution, and zone-based for data locality.

3. Different types of indexes in MongoDB

MongoDB supports several types of indexes:

// Single Field Index
db.users.createIndex({ email: 1 });

// Compound Index
db.users.createIndex({ lastName: 1, firstName: 1 });

// Text Index
db.articles.createIndex({ content: "text" });

// Geospatial Index
db.locations.createIndex({ coordinates: "2dsphere" });

// Hashed Index
db.collection.createIndex({ _id: "hashed" });

Each index type optimizes different types of queries. Choose based on your query patterns.

4. Embedded documents in MongoDB

Embedded documents are documents nested inside other documents, creating a parent-child relationship.

db.users.insertOne({
    name: "John Doe",
    address: {
        street: "123 Main St",
        city: "Boston",
        state: "MA",
        zip: "02108"
    },
    contacts: [
        { type: "email", value: "john@example.com" },
        { type: "phone", value: "555-0123" }
    ]
});

Embedded documents provide better data locality and are ideal for one-to-few relationships.

5. Git tags and their types

Git tags are references to specific points in Git history, typically used for marking release points.

Lightweight tags:

git tag v1.0.0

Annotated tags:

git tag -a v1.0.0 -m "Release version 1.0.0"

Lightweight tags are just pointers to commits, while annotated tags are full objects containing tagger name, email, date, and message.

6. Git Reflog vs Git Reset

Git Reflog:

  • Records all reference updates in the local repository
  • Acts as a safety net for recovering lost commits
  • Only exists locally

Git Reset:

  • Moves the HEAD and branch pointer to a specific commit
  • Can modify working directory and staging area
  • Different modes: --soft, --mixed, --hard

While reset changes branch state, reflog helps recover from mistakes made with reset or other commands.

7. Git Rebase vs Git Merge

Git Merge:

  • Creates a new merge commit
  • Preserves complete history
  • Better for public branches

Git Rebase:

  • Moves commits to a new base
  • Creates linear history
  • Better for private/local branches

Merge maintains history but can create complex graphs. Rebase creates cleaner history but should not be used on public branches.

Long Questions

1. Social Media Database Operations

// a) Insert documents
db.posts.insertMany([
    {
        content: "Learning MongoDB basics",
        author: "john_doe",
        likes: 120,
        tags: ["technology", "database", "learning"],
        createdAt: new Date("2024-01-01")
    },
    {
        content: "Web Development Tips",
        author: "jane_smith",
        likes: 85,
        tags: ["technology", "webdev"],
        createdAt: new Date("2024-01-15")
    }
    // ... more documents
]);

// b) Find posts with technology tag
db.posts.find(
    { tags: "technology" }
).sort({ likes: -1 });

// c) Update posts with >100 likes
db.posts.updateMany(
    { likes: { $gt: 100 } },
    { $set: { trending: true } }
);

// d) Average likes per author
db.posts.aggregate([
    {
        $group: {
            _id: "$author",
            avgLikes: { $avg: "$likes" }
        }
    }
]);

// e) Delete old posts
db.posts.deleteMany({
    createdAt: {
        $lt: new Date(Date.now() - 30*24*60*60*1000)
    }
});

2. Blog Database Operations

// a) Insert articles
db.articles.insertMany([
    {
        title: "JavaScript Best Practices",
        content: "Learn about modern JavaScript...",
        author: "john_doe",
        tags: ["javascript", "programming"],
        createdAt: new Date()
    }
    // ... more documents
]);

// b) Create text index
db.articles.createIndex({
    title: "text",
    content: "text"
});

// c) Find JavaScript articles
db.articles.find(
    { $text: { $search: "JavaScript" } }
).sort({ createdAt: -1 });

// d) Update database articles
db.articles.updateMany(
    { content: /database/i },
    { $set: { category: "Database" } }
);

// e) Delete articles without tags
db.articles.deleteMany({
    $or: [
        { tags: { $exists: false } },
        { tags: { $size: 0 } }
    ]
});

3. Git Open Source Project Commands

# a) Set remotes
git remote add origin https://github.com/yourusername/project.git
git remote add upstream https://github.com/original/project.git

# b) Fetch upstream branches
git fetch upstream

# c) Create feature branch
git checkout -b feat6 upstream/feat6

# d) Stage and commit
git add .
git commit -m "Completed Task 6"

# e) Push to fork
git push origin feat6

4. Git Commit History Management

# a) View commit history
git log --oneline -n 4

# b) Start interactive rebase
git rebase -i HEAD~4

# c) In the interactive rebase editor:
# pick abc1234 First commit message
# squash def5678 Second commit message
# squash ghi9012 Third commit message
# squash jkl3456 Fourth commit message

# d) Force push
git push origin feature-branch -f

# e) Verify history
git log --oneline
